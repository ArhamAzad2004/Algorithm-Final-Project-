① OPT ≥ MST (why?)

Remove any one edge from the optimal tour → you get a spanning tree (all cities still connected). MST is the minimum weight spanning tree. So MST ≤ any spanning tree ≤ OPT.

② DFS walk = 2 × MST (why?)

A DFS traversal of the MST visits every edge exactly twice — once going down the tree, once returning up. Total DFS walk cost = 2 × MST weight.

③ Approx tour ≤ DFS walk (why?)

Shortcutting skips already-visited cities. By the triangle inequality: d(A,C) ≤ d(A,B) + d(B,C). Each shortcut costs no more than the detour. So the tour is at most the DFS walk cost.

🔑 Therefore: Approx tour ≤ DFS ≤ 2×MST ≤ 2×OPT

This ONLY holds for metric TSP (triangle inequality must hold). Without it, shortcutting can make the tour worse and the proof breaks completely.

📐 Fully Worked Example: 4-City Square

▼

Cities: A(0,0), B(1,0), C(1,1), D(0,1). All edges are Euclidean distances.

🔑 Step 1: Build MST (Prim's starting at A)

Add edge A-B (cost 1.0). Add edge B-C (cost 1.0). Add edge C-D (cost 1.0). MST weight = 3.0. Note: diagonal A-C would cost √2 ≈ 1.414, so sides are cheaper.

🔑 Step 2: DFS walk on MST

Starting at A: A→B→C→D→C→B→A (traverses every edge twice). DFS walk cost = 6.0 = 2 × 3.0 ✓

🔑 Step 3: Shortcut DFS walk

DFS order (first visit): A, B, C, D. Tour: A→B→C→D→A. Cost = 1+1+1+1 = 4.0. Note: 4.0 ≤ 6.0 (DFS walk) ≤ 2×3.0=6.0. Ratio = 4/OPT = 4/4 = 1.0× (happens to be optimal here).

💡 Key Concepts

▼
🔑 Why is TSP NP-complete?

No polynomial-time exact algorithm is known. For n cities, brute force checks (n−1)!/2 tours. For n=20: over 60 trillion tours. Approximation algorithms give us good solutions in polynomial time.

🔑 Triangle Inequality — WHY it's essential

d(A,C) ≤ d(A,B) + d(B,C) for all cities. Euclidean distance always satisfies this. Without it, the shortcutting step (step 3) would be invalid — a direct edge could cost MORE than the detour, and the proof breaks.

🔑 MST is a lower bound on OPT

Take any Hamiltonian tour (visiting all cities once). Remove one edge → spanning tree. MST ≤ this spanning tree ≤ OPT. Therefore MST ≤ OPT always.

🔑 Christofides: 3/2-Approximation

Better algorithm: Build MST, then find a minimum weight perfect matching on odd-degree vertices, combine to get an Eulerian circuit, then shortcut. Guarantees tour ≤ 1.5×OPT. More complex but closer to optimal.

🔑 When does 2× approximation = 1×?

When the MST itself forms a Hamiltonian path (visiting all cities without revisiting). Then DFS walk = 2×MST but shortcutting perfectly recovers the MST path, and tour = MST weight = OPT.

📖 Lecture Notes — 2-Approximation for Metric TSP

▼
1. The Traveling Salesman Problem

Given n cities with pairwise distances, find the shortest tour visiting each city exactly once and returning to start. NP-complete: no polynomial-time exact solution known. For n=20: (n−1)!/2 ≈ 6×10¹⁶ tours.

2. Metric TSP and Triangle Inequality

In metric TSP, distances satisfy the triangle inequality: d(u,w) ≤ d(u,v) + d(v,w) for all cities. Euclidean distance (on a map) always satisfies this. This constraint allows the 2-approximation — without it, the problem is even harder to approximate.

3. The Algorithm
APPROX-TSP(G): 1. T = MST(G) // minimum spanning tree 2. W = DFS-WALK(T) // preorder DFS traversal 3. Tour = SHORTCUT(W) // skip revisited cities 4. return Tour
4. Prim's MST Algorithm
PRIM(G, start): inMST = {start} while |inMST| < n: (u,v) = min-weight edge with u∈inMST, v∉inMST add edge (u,v) to MST add v to inMST

Greedily adds the cheapest edge that connects a new city to the growing MST. O(n²) with adjacency matrix, O(m log n) with priority queue.

5. Why DFS Walk = 2 × MST

A DFS traversal on a tree visits every edge exactly twice: once in each direction (forward when going deeper, backward when returning). So the total walk cost = sum of all MST edge weights × 2 = 2 × MST weight.

6. Shortcutting with Triangle Inequality

When DFS revisits a city, we skip it and go directly to the next unvisited city. Triangle inequality guarantees this direct edge costs ≤ the detour. Therefore: Tour cost ≤ DFS walk cost.

7. Christofides (3/2-Approximation)

Build MST, identify odd-degree vertices (always even count by handshaking lemma), find minimum weight perfect matching on them, combine MST + matching edges into an Eulerian graph, find Eulerian circuit, shortcut. This gives 3/2 × OPT guarantee. Published 1976 — best known ratio for over 40 years until 2020.

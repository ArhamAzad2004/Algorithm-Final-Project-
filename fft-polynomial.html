📐 Fully Worked Example: (1 + 2x + 3x²) × (4 + 5x)

▼

We multiply A(x) = 1 + 2x + 3x² by B(x) = 4 + 5x. The expected result is 4 + 13x + 22x² + 15x³.

🔑 Key Idea: Why Pad to n=4?

A has degree 2, B has degree 1. Product has degree 3 → needs 4 coefficients → next power of 2 ≥ 4 is 4. We pad both to length 4.

🔑 Butterfly at Level 1, index 0 (n=4):

t = ω⁰ · y_odd[0] = 1 · 2 = 2. Upper = y_even[0] + t = 1+2 = 3. Lower = y_even[0] − t = 1−2 = −1. This is one butterfly: 1 multiplication + 2 additions.

💡 Key Concepts

▼
🔑 Cancellation Lemma

ω^(n/2) = −1, so ω^(k + n/2) = −ω^k. This means the twiddle factors for the second half of a butterfly level are the negatives of the first half — allowing massive cancellation and enabling the divide-and-conquer split.

🔑 Halving Lemma

Squaring the n roots of unity gives exactly the n/2 roots of unity for the next level. This is WHY we can split a DFT of size n into two DFTs of size n/2 — the evaluation points halve perfectly.

🔑 Why O(n log n)?

Recurrence: T(n) = 2T(n/2) + O(n). Each level does O(n) butterfly ops. There are log₂(n) levels. Total = O(n log n). Identical to merge sort's recurrence — divide by 2, then O(n) combine work.

🔑 IFFT is nearly identical to FFT

Replace ω with ω⁻¹ = e^(−2πi/n) and divide each output by n. The DFT matrix is Vandermonde; its inverse is Vandermonde with conjugated entries scaled by 1/n. Remarkable symmetry.

1. The Polynomial Multiplication Problem

Given two polynomials A(x) = a₀ + a₁x + … + aₙ₋₁xⁿ⁻¹ and B(x) = b₀ + b₁x + … + bₙ₋₁xⁿ⁻¹, compute their product C(x) = A(x)·B(x). Coefficients of C are given by the convolution: cₖ = Σ aᵢ · bₖ₋ᵢ. This appears everywhere: signal processing, audio filtering, cryptography, number multiplication.

2. Naïve O(n²) Approach
NAIVE-MULTIPLY(A, B): C = zeros(2n-1) for i = 0 to n-1: for j = 0 to n-1: C[i+j] += A[i] * B[j] // n² mults return C

Every output coefficient C[k] requires up to n multiplications. For n=10⁶, that is 10¹² operations — completely infeasible.

3. Roots of Unity — The Key Ingredient

The n-th roots of unity are ω⁰, ω¹, …, ωⁿ⁻¹ where ω = e^(2πi/n). They are evenly spaced on the unit circle. Two critical properties enable FFT:

🔑 Cancellation Lemma: ω^(n/2) = −1

The point at angle π is −1. So ω^(k+n/2) = −ω^k. Upper/lower butterfly outputs share one twiddle factor → two outputs for the cost of one multiplication.

🔑 Halving Lemma: (ωₙ)² = ωₙ/₂

Squaring maps n roots to n/2 roots. A DFT of size n splits into two DFTs of size n/2 at different points. This is the recursive structure of FFT.

4. The FFT Idea — Evaluate, Don't Convolve
Step 1 — EVALUATE: FA = FFT(A, 2n); FB = FFT(B, 2n) [O(n log n)] Step 2 — POINTWISE: FC[k] = FA[k] * FB[k] for each k [O(n)] Step 3 — INTERPOLATE: C = IFFT(FC, 2n) [O(n log n)]

A polynomial of degree n−1 is uniquely determined by its values at n points. So instead of multiplying coefficients directly, we evaluate at 2n special points (roots of unity), multiply values pairwise, then recover coefficients via IFFT.

5. Butterfly Operations
BUTTERFLY(a, b, ω_k): t = ω_k * b // one complex multiply upper = a + t // one add lower = a - t // one subtract return (upper, lower)

The butterfly is named because its data flow diagram looks like wings crossing. It is the core O(1) unit. There are (n/2)·log₂(n) butterflies total. Each level of the recursion has n/2 butterflies; there are log₂(n) levels.

7. Inverse FFT
IFFT(Y): result = FFT(Y, use ω⁻¹ = e^(-2πi/n)) return result / n // divide every element by n

The IFFT is structurally identical to the FFT. Replace ω with its conjugate ω⁻¹ and scale by 1/n at the end. This works because the DFT matrix is invertible with a conjugate-transpose inverse (scaled).

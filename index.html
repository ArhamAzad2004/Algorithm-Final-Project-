FFT Polynomial Multiplication

Divide-and-conquer in the frequency domain: forward FFT, pointwise multiply, inverse FFT. O(n log n) instead of O(n²).

Live: 8 roots of unity (e^(2πik/8))
n=8 → FFT: ~24 ops · Naive: 64 ops · 2.7× faster
Used everywhere fast convolution is needed: digital signal processing, audio/image filtering, large integer multiplication, computer algebra systems. Going from O(n²) to O(n log n) makes things possible that brute force can't touch.

CLO-3 ✓ Visited

Miller-Rabin Primality Testing

Probabilistic primality test using witness rounds. The basis of every RSA key generator on Earth.

Live tester — try 561 (Carmichael) or 104729
→ click Test to run 5 witness rounds
Generating an RSA key needs two ~1024-bit primes. Trial division would take longer than the age of the universe; Miller-Rabin gives a probability-of-error of 4⁻ᵏ in O(k · log³n) — done in milliseconds.
Open full tool → Press 2 to open
CLO-4 ✓ Visited

Rabin-Karp String Matching

Find a pattern in a string using a rolling hash that updates in O(1) per slide. Expected O(n + m).

Live matcher — pattern is highlighted in real time
→ live highlights appear here
Search-in-large-text use cases: plagiarism detection, network intrusion signatures, DNA pattern matching, the Ctrl+F in your editor. Rolling hash also extends naturally to 2D (image patches) and to multi-pattern search.
Open full tool → Press 3 to open
CLO-5 ✓ Visited

2-Approximation for Metric TSP

TSP is NP-complete, but for metric instances we can guarantee a tour at most 2× optimal using MST + DFS + shortcutting.

Click on the map to place cities (max 7) — tour redraws instantly
→ click 3+ points above
Real applications: vehicle routing, PCB drilling, warehouse pick-paths. The 2-approximation guarantee means even on a thousand cities you're within 2× of optimal — and you computed it in polynomial time.

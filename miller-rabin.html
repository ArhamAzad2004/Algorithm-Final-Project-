📐 Fully Worked Example: n = 221, witness a = 2

▼

We test n = 221 = 13 × 17 (composite) with witness a = 2.

🔑 Step 1: Decompose n−1 = 2ˢ · d

n−1 = 220. Divide by 2 repeatedly: 220 → 110 → 55 (odd). So s = 2, d = 55. We write 220 = 2² × 55.

🔑 Step 2: Compute a^d = 2^55 mod 221 via Square-and-Multiply

Binary of 55 = 110111. Process each bit: start result=1, base=2. For each '1' bit: result = result × base mod 221. For each bit: base = base² mod 221. Result = 137.

🔑 Why is this COMPOSITE?

We squared up to a^(n−1) = 1. But we never saw −1 (= 220 mod 221) on the way. This means we found a non-trivial square root of 1: 47² ≡ 1 (mod 221) but 47 ≢ ±1. Only possible in composite rings. QED: 221 is composite.

💡 Key Concepts

▼
🔑 Non-Trivial Square Root of 1

In prime fields, x² ≡ 1 (mod p) implies x ≡ ±1. In composite rings, other solutions exist. Finding one (a number that squares to 1 but isn't ±1) proves compositeness. This is Miller-Rabin's core insight that Fermat lacks.

🔑 Why n−1 = 2ˢ · d?

Since n is odd, n−1 is even. Factoring out all 2s gives us s squarings to check. Each squaring is a chance to find a non-trivial √1. The chain a^d → a^(2d) → … → a^(n−1) must hit −1 at some point if n is prime.

🔑 Error Probability = 4⁻ᵏ

For any composite n, at least 3/4 of all possible witnesses will detect it in one round. After k rounds with random witnesses: probability of false "prime" ≤ (1/4)^k. At k=40: probability ≈ 10⁻²⁴ — negligible.

🔑 Carmichael Numbers (561, 1105, 1729…)

These composite numbers satisfy a^(n−1) ≡ 1 for ALL valid witnesses a. Fermat's test is completely fooled. Miller-Rabin detects them because even though a^(n−1) ≡ 1, the squaring chain reveals a non-trivial √1 along the way.

🔑 Square-and-Multiply is O(log d)

Computing a^d mod n naïvely needs d multiplications. Square-and-multiply processes one bit of d per iteration: square always, multiply if bit = 1. For a 2048-bit exponent: ~2048 squarings vs 2^2048 naïve multiplications.

📖 Lecture Notes — Miller-Rabin Primality Testing

▼
1. Why Primality Testing Matters

RSA encryption requires two large random primes p and q (1024–4096 bits each). Naïve trial division up to √n is completely infeasible — √(2²⁰⁴⁸) ≈ 2¹⁰²⁴, more atoms than the observable universe. Miller-Rabin tests in O(k · log²n) time with error ≤ 4⁻ᵏ.

2. Fermat's Little Theorem & Its Limits

If p is prime: a^(p−1) ≡ 1 (mod p) for all a not divisible by p. The Fermat test checks this — but Carmichael numbers fool it completely. Miller-Rabin fixes this by additionally checking the squaring chain.

3. Decomposing n−1 = 2ˢ · d
DECOMPOSE(n): d = n - 1; s = 0 while d is even: d = d / 2; s = s + 1 return (s, d) // n−1 = 2^s * d, where d is odd
4. The Squaring Chain
MILLER-RABIN-WITNESS(n, a, s, d): x = modpow(a, d, n) // square-and-multiply if x == 1 or x == n-1: return PROBABLY_PRIME for r = 1 to s-1: x = x*x mod n // square if x == n-1: return PROBABLY_PRIME return COMPOSITE // hit 1 without −1

Three cases: (1) a^d ≡ 1 → passes immediately. (2) Some a^(2ʳd) ≡ −1 → passes. (3) Chain reaches 1 without −1 → non-trivial √1 found → COMPOSITE.

6. Square-and-Multiply: O(log d)
MODPOW(base, exp, mod): result = 1; base = base mod mod while exp > 0: if exp is odd: result = result * base mod mod // multiply exp = exp >> 1 // halve exponent base = base * base mod mod // square return result

Each iteration halves the exponent. Total iterations = ⌊log₂ d⌋ + 1. For d = 2048-bit number: ~2048 iterations vs 2^2048 naive multiplications.
